# Lessons Learned

*Add dated entries as patterns and gotchas are discovered.*

## 2026-04-28

- **`portal_docker_socket_proxy` healthcheck stuck the watchdog in a 5-min restart loop for hours.** The compose healthcheck used `exec 3<>/dev/tcp/localhost/2375 && echo ok` — bash's `/dev/tcp` is a built-in, not a real device. The `tecnativa/docker-socket-proxy` image runs BusyBox `ash` as `/bin/sh`, which doesn't implement that magic. Every healthcheck failed with `sh: can't create /dev/tcp/localhost/2375: nonexistent directory` even though the proxy itself was answering `/_ping` perfectly. Watchdog spammed Mattermost every 5 min restarting it forever. Fix in `docker-compose.yml` (commit `8309d18`): `test: ["CMD", "sh", "-c", "wget -q -O- http://localhost:2375/_ping || exit 1"]` — wget is in BusyBox, and `/_ping` actually verifies the proxy responds (stronger than a port-listen check). Codified as Arbiter Law 013 (`config/013-no-dev-tcp-healthcheck.md`) so this can never silently happen again on a fleet repo. Also: when adding any healthcheck, test it inside the actual image's shell first — `docker exec <container> sh -c '<your test>'` and verify exit 0.
- **Toshi's local `~/BirdMug-Portal` repo had `origin = https://github.com/...`, not `git@github.com:...`.** Pushing over HTTPS from an SSH session fails because no credential helper is available — `fatal: could not read Username for 'https://github.com'`. Other repos on Toshi use `git@github.com:falkensteink/<repo>.git`. Fixed with `git remote set-url origin git@github.com:falkensteink/BirdMug-Portal.git`. Going forward, any new repo cloned to Toshi should use SSH remote from the start.
